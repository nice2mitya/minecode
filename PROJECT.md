# MineCode.ru — Project Bible

## Миссия
Бесплатный курс программирования для школьников 6-9 классов (11-15 лет) через вселенную Minecraft. Не "программирование для малышей" — серьёзный подход к computational thinking, архитектуре и AI-эре, упакованный в увлекательный формат.

## Целевая аудитория
- **Первичная:** Лукьян (6 класс, высокий IQ, начитанный) — тестировщик #1, со-разработчик
- **Вторичная:** Леон (9 класс) — тестировщик #2
- **Широкая:** любой школьник 6-9 класса (11-15 лет), начинающий в программировании
- **Тон:** умный, не снисходительный, с юмором, как разговор со старшим братом-программистом

## Философия 2026
- Через год писать код руками почти никто не будет — учим ДУМАТЬ, а не печатать синтаксис
- Computational thinking > конкретный язык
- Архитектура, паттерны, системное мышление — вечные навыки
- AI как инструмент (как калькулятор в математике) — учим работать С ним
- Vibe coding — легитимный подход, учим делать это осознанно

## Геймификация
- Duolingo-подобная система: XP, уровни, стрики, ачивки
- Minecraft как метафора: блоки кода = блоки в игре
- Каждый урок = мини-квест с конкретным результатом в мире Minecraft
- Прогресс виден визуально (строишь город/базу по мере прохождения)

## Участие Лукьяна
- Не просто ученик, а **бета-тестер и со-продюсер**
- После каждого урока: что понравилось, что скучно, что непонятно
- Его фидбек = приоритетный input для итерации
- Можно дать ему роль "первого ревьюера" — это повышает engagement

## Production Pipeline (автономный)

### Роли (все выполняет AI, но с чёткой специализацией)
1. **Методист** — структура урока, learning objectives, pedagogy
2. **Копирайтер** — текст урока, диалоги, нарратив
3. **Редактор** — вычитка, стиль, tone of voice, factcheck
4. **Дизайнер** — визуальная концепция, UI/UX, иллюстрации
5. **Верстальщик** — HTML/CSS реализация дизайна
6. **Программист** — интерактивные элементы, code editor, логика
7. **DevOps** — деплой, CI/CD, мониторинг
8. **Тестер** — проверка всех интерактивов, кросс-браузер
9. **Критик-продюсер** — финальная проверка, может отправить на доработку

### Цикл создания урока
```
[Методист] → структура + LO
    ↓
[Копирайтер] → текст + нарратив
    ↓
[Редактор] → правка + factcheck
    ↓
[Дизайнер] → макет + визуалы
    ↓
[Верстальщик] → HTML/CSS
    ↓
[Программист] → интерактивы
    ↓
[Тестер] → проверка
    ↓
[Критик-продюсер] → approve / → на доработку (любому из этапов)
    ↓
[DevOps] → деплой на minecode.ru
    ↓
[Лукьян + Леон] → фидбек → итерация
```
