Estructuras anidadas: un mundo de fragmentos

Construyendo datos complejos: una lista de diccionarios y un diccionario de listas.

🎣 Hook

Un fragmento de Minecraft contiene un montón de bloques. Un mundo: un montón de trozos. Los sistemas complejos se componen de sistemas anidados.

🧠 Concept

Términos: lista, diccionario, bucle, función: escribimos en inglés, explicamos en ruso.

Las estructuras anidadas ocurren cuando hay diccionarios dentro de una lista y listas dentro de un diccionario.

Ejemplo:
trozo = {
"coordenada": [0, 0],
"recursos": ["carbón", "hierro"],
"mobs": [{"tipo": "zombis", "hp": 20}]
}

Es conveniente transferir estructuras como JSON y almacenar el estado del mundo.

🔧 Practica

Tarea 1: agregue una clave para la lista de recursos y un índice para el primer recurso.

trozo = {"recursos": ["carbón", "hierro"]}
imprimir(fragmento["recursos"][0])

Complete los espacios en blanco del código en su cuaderno/editor. Respuestas clave: recursos/0.

Tarea 2: indica la clave del campo hp para el primer mob.

trozo = {"mobs": [{"tipo": "zombi", "hp": 20}]}
imprimir(fragmento["mobs"][0]["hp"])

Ponte a prueba: hp.
