Leer código de IA, como un revisor

Verificamos el código de IA en busca de casos lógicos, de seguridad y extremos.

🎣 Hook

La IA puede escribir código funcional, pero se pierde un detalle y todo se estropea. Tu superpoder: revisión y verificación.

🧠 Concept

Términos: lista, diccionario, bucle, función: escribimos en inglés, explicamos en ruso.

Lista de verificación de revisión:
1) ¿el código resuelve el problema?
2) ¿Se procesan casos extremos?
3) legibilidad y nombres?
4) ¿Hay lugares peligrosos?
5) ¿hay pruebas?

La IA acelera el borrador, pero la calidad final es tu responsabilidad.

🧩 Prueba +10 XP
¿Qué es lo que la IA suele pasar por alto sin una petición explícita?
A Comentarios musicales
B Casos límite
C El acolchado siempre es perfecto.
D Nombre de archivo

🔧 Practica

Tarea 1: elige el primer paso de verificación.

paso = "verificar con requisito"

Complete los espacios en blanco del código en su cuaderno/editor. Respuestas clave: requisito.

Tarea 2: agregar el tipo de pruebas para entradas no estándar.

tests = ["happy", "edge", "error"]

Ponte a prueba: ventaja.

🧩 Prueba +10 XP
La frase "código se ejecuta" significa que...
A La decisión es definitivamente correcta.
B El código ya está disponible para producción.
C Sólo que no hay ningún error de inicio instantáneo.
D No se necesitan pruebas

🏗️ Montaje: mini proyecto

Tome un fragmento de código de IA (cualquier ejemplo) y haga comentarios de revisión: 3 fortalezas, 3 riesgos, 2 ediciones.

👹 Mini jefe +120 XP

Combinamos el tema actual con habilidades previas.

A Hazlo sin estructura y luego arréglalo.
B Primero planificar, luego implementar y probar
C Copiar la solución sin entender.
D Saltar pruebas de velocidad
