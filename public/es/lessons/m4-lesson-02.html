Ingeniería rápida: hable con la IA con precisión

Aprendemos a plantear un problema de IA para que la respuesta sea verificable y útil.

🎣 Hook

El comando /dar en Minecraft sólo funciona si los parámetros son precisos. Lo mismo ocurre con la IA: solicitud poco clara = resultado poco claro.

🧠 Concept

Términos: lista, diccionario, bucle, función: escribimos en inglés, explicamos en ruso.

Un buen mensaje incluye:
- rol/contexto;
- objetivo específico;
- datos de entrada;
- formato de respuesta;
- criterios de verificación.

No "hacerlo hermoso", sino "ofrecer 3 opciones de API con ventajas y desventajas".

🧩 Prueba +10 XP
¿Qué suele degradar una respuesta de IA?
A Restricciones claras
B Declaración de problema incierto
C Formato de salida explícito
D Contexto del proyecto

🔧 Practica

Tarea 1: agregue el bloque "formato de respuesta" requerido.

Prompt = "... formato: tabla de 3 puntos"

Complete los espacios en blanco del código en su cuaderno/editor. Respuestas clave: formato.

Tarea 2: indicar el requisito de verificación.

mensaje += "+ comprobando casos extremos"

Ponte a prueba: compruébalo.

🧩 Prueba +10 XP

A Sólo "más rápido"

🏗️ Montaje: mini proyecto

Reescriba el mensaje débil "crear un sistema de creación" en uno fuerte: requisitos, interfaz, casos de prueba, restricciones.
