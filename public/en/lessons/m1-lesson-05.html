Functions - create your own teams

I wrote once - call as many times as you want.

🎣 Start

You already know how to build a wall. What if you need 10 different walls in different places? Copy the code 10 times?

No. You create a function - your own team. I wrote it once, call as many times as you want.

A function is like a crafting recipe. Once you write down the pickaxe recipe, you can craft as many pickaxes as you like without having to remember the location of the ingredients each time.

🧠 Theory: Functions

def name(): - create a function

Keyword def (from “define” - define) creates a new command:

def build_a house():
    put up_walls()
    put_the_roof on()
    put_the_door()

Now build_house() - your personal team. I wrote once - you build houses everywhere:

build_a house()   # house number 1
forward()
build_a house()   # house number 2
forward()
build_a house()   # house number 3

Settings - Customizable Features

Do you want walls of different sizes? Add parameters:

def build_a wall(width, height):
    for string in range(height):
        for block in range(width):
            put_block()

build_a wall(10, 5)   #wall 10×5
build_a wall(3, 20)    # tower 3×20

Parameters are like ingredients in a recipe. Different ingredients → different results.

return — function result

A function can not only do something, but also return result:

def area(a, b):
    return a*b

result = area(10, 5)   # result = 50

Analogy: function = crafting recipe. Parameters = ingredients. return = crafting result.
