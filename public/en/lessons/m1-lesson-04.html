Cycles - build a wall in 3 seconds

Why write the same thing 100 times if you can repeat it once?

🎣 Start

Do you remember how you wrote forward() five times in a row in the first lesson? Imagine that you need to build a wall of 100 blocks. write forward() a hundred times?

No. There are cycles for this.

A cycle is a “repeat” command. One line instead of a hundred. Programmers are lazy - and this is their superpower.

🧠 Theory: For Loop

for i in range(n) — repeat n times

The most common loop in Python. You say “how many times to repeat” - and he repeats.

for i in range(10):
    put_block()
    forward()

Minecraft analogy: place 10 blocks in a row. Without a loop - 20 lines. With cycle - 3.

range() - number generator

range(5) gives the numbers: 0, 1, 2, 3, 4

⚠️ Starts from scratch! And it ends up to the specified number. range(5) - these are 5 numbers, but the last one is 4, not 5.

Loop variable i

i - this is a counter. At each step of the cycle it increases. Like the number of the block you are placing.

for i in range(5):
    print(i)    # will output: 0, 1, 2, 3, 4

Nested loops are a wall!

Wall = rows × columns. Loop within a loop:

for string in range(5):       # 5 lines
    for block in range(10):   # 10 blocks each
        put_block()
        forward()

Total: 5 × 10 = 50 blocks. Two lines of code instead of fifty.

🔧 Practice: Building in cycles

Task 1: Build a line of 7 blocks.

for i in range():
    put_block()
    ()
💡 Hint

You need 7 blocks - that means range(???). After each block, the turtle takes a step forward.

✅ Solution 
for i in range(7):
    put_block()
    forward()

Task 2: Build a wall 4 height × 6 width (nested loop).

for string in range():
    for block in range():
        put_block()
        forward()
    # move to next row
💡 Hint

Height = number of rows (outer loop). Width = number of blocks in each line (inner loop).

✅ Solution 
for string in range(4):
    for block in range(6):
        put_block()
        forward()
    # move to next row

🏗️ Build: The Wall in Action

Imagine: you wrote 3 lines of code, clicked “run” - and the turtle itself built a wall of 24 blocks. Without a single manual click.

This is what loops do: turn minutes of manual work into seconds of automatic. It's like the difference between digging for diamonds with your hands and using a pickaxe with Efficiency V.

👹Boss: Pixel Art +75 XP

The turtle draws a 5x5 square from blocks. Fill in the blanks:

for string in range():
    for block in range():

        forward()
    # return to the beginning of the row and go up
    ()
    for i in range(5):

    ()

💡 Hint

5x5 square - both range() are the same. After each row, the turtle turns around, goes back, turns around again and rises one block up.

✅ Solution 
for string in range(5):
    for block in range(5):
        put_block()
        forward()
    turn around()
    for i in range(5):
        forward()
    turn around()
    up()
