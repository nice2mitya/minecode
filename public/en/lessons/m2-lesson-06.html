Modularity - Redstone blocks

We break the system into independent parts with clear interfaces.

🎣 Hook

One redstone mechanism is easy to break. A system of individual components is easier to repair and expand.

🧠 Concept

Terms: list, dictionary, loop, function - we write in English, we explain in Russian.

Modularity = we divide the project into components, each responsible for its own task.

Principles:
- one responsibility per module;
- clear interface (what it accepts/what it returns);
- minimum hidden dependencies.

This is how the project grows without chaos.
