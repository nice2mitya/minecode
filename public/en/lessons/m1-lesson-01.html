What is the code

And why are you already a programmer, even if you don’t know about it.

🎣 Start

Open Minecraft. Craft a stone pickaxe.

Did you do it? OK. You have just completed the program. No, seriously.

Look:

1. Open the workbench
2. Place 3 cobblestones in the top row
3. Place 1 stick in the center
4. Place 1 stick in the bottom center
5. Take the pickaxe

Five steps. Clear sequence. Specific result. If you change the order or ingredients, you will get something different or nothing at all.

This is the program - a set of instructions that are executed in order and produce a result.

Congratulations, you are already thinking like a programmer. All that remains is to learn how to write these instructions so that the computer understands them.

🧠 Theory: Algorithm

Algorithm is a sequence of steps to solve a problem.

You use algorithms every day:

  • Crafting recipe = algorithm
  • Route to school = algorithm
  • Strategy against the Ender Dragon = algorithm (difficult)

Programming is when you write an algorithm in a language that the computer understands.

Example: building a line of 5 blocks

In words:

Put up a block. Take a step forward. Put up a block. Take a step forward. Put up a block. Take a step forward. Put up a block. Take a step forward. Put up a block.

In short (with a loop - but that's lesson 4):

Repeat 5 times: place a block, step forward.

On the code:

for i in range(5):
    put_block()
    step_forward()

It is not necessary to understand every line yet. The main thing is to see how verbal instructions turn into code? It's the same algorithm, just written differently.

Three properties of a good algorithm

  1. Accuracy — every step is unambiguous. “Put a block” - understandable. “Make something beautiful” - no.
  2. Order — steps are performed sequentially. You cannot pick up the pickaxe before you have placed the ingredients.
  3. Limb — the algorithm ends. If your program takes forever, it's a bug.

🏗️ Build: Turtle in Action

We run your algorithm in the virtual world. The turtle follows your commands and leaves blocks.

This is what programming is: wrote → ran it → saw the result → corrected it → ran it again. This cycle is called debug loop, and mistakes are a normal part of the process.
