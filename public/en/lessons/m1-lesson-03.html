Conditions - if creeper then run

How to teach a program to make decisions.

🎣 Start

You're walking through a cave in Minecraft. There's a mob ahead. What are you doing?

  • If this cow → ignore
  • If this zombie → you hit with a sword
  • If this creeper → RUN 💨
  • If this Enderman → don’t look him in the eyes

You've just met four conditions. Your brain does this automatically: it evaluates the situation and chooses an action. Programs work the same way - but they need to specify each condition explicitly.

🧠 Theory: if/else

Condition - This is a fork in the code. If something is true, we do one thing. If not, something else.

mob = "creeper"

if mob == "creeper":
    print("LET'S RUN!")
else:
    print("Let's attack!")

Let's break it down piece by piece:

  • if - “if”
  • mob == "creeper" — check (two characters = - this is a comparison, one = - assignment)
  • : — colon after the condition (Python wants it this way, don’t argue)
  • Indentation (4 spaces) - what is done if the condition is true
  • else: - “otherwise” (everything else)

elif - when there are more than two options

mob = "zombie"

if mob == "creeper":
    print("LET'S RUN!")
elif mob == "zombie":
    print("We hit with a sword")
elif mob == "skeleton":
    print("Hide behind the shield")
elif mob == "enderman":
    print("Don't make eye contact")
else:
    print("Something unknown. Be careful.")

elif is an abbreviation for “else if”. You can bet as much as you like.

Task 1: Mob classifier

Into empty fields ??? enter the commands: if (if) elif (and if) or else (otherwise). These are the words that tell the program when to execute the code.

mob = "wolf"

 mob == "creeper":
    type = "hostile"
 mob == "wolf":
    type = "neutral"
:
    type = "friendly"
💡 Hint

The first check is if. Additional check - elif. Everything else is else.

✅ Solution 
if mob == "creeper":
    type = "hostile"
elif mob == "wolf":
    type = "neutral"
else:
    type = "friendly"

🏗️ Build: Smart Turtle

Remember the turtle from lesson 1? She walked forward stupidly. Now she will become smarter.

The turtle moves around the world and encounters different blocks. She can determine them using the function what_ahead().

block = what's ahead()

if block == "air":
    forward()
elif block == "ore":
    dig()
    forward()
elif block == "lava":
    left()
    forward()
    right()
    forward()
    right()
    forward()
    left()
elif block == "bedrock":
    print("We can't go any further")
    turn around()

Turtle now reacts on the environment, and does not just blindly follow commands. This is already real AI (well, almost).

💭 Results

Now you know:

  • ✅ if/elif/else - the program can make decisions
  • ✅ Comparison operators: ==, !=, >, <, >=, <=
  • ✅ Nested conditions - decision tree

Where is it in the real world:

  • Mail spam filter: if contains “you won a million” → spam
  • Automatic doors: if motion sensor → open
  • Traffic light: if red → stop, elif green → go

Fun fact: All logic of any computer comes down to two operations: comparison (if) and transition (goto). Literally everything. Games, neural networks, operating systems - just a lot if. 🤯
